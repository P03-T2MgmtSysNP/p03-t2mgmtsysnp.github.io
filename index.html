Project Overview

The goal of this project is to design and prototype a low-cost, user-friendly management system for the Georgia Laws of Life Essay Contest. The system needs to consolidate data that currently lives in multiple spreadsheets into a single CRM environment where sponsors, volunteers, donors, and communications can be tracked efficiently.

Problem

  • Data spread across many spreadsheets.
  • Limited visibility into relationships between schools, sponsors, and contacts.
  • Manual processes for communication and reporting.

Solution Approach

  • Evaluate multiple CRM platforms against client needs.
  • Select a low-code, scalable CRM (HubSpot) as the primary platform.
  • Design data models, integrations, and workflows for the client’s context.
  • Implement and test a proof-of-concept configuration.

Semester Timeline

The following phases show how the project evolved from initial planning to the final package delivered to our client.

1. Project Planning

At the beginning of the semester, we defined the scope, stakeholders, constraints, risks, and schedule for the Management System for Nonprofits project. A detailed Gantt chart was created to estimate effort, sequence tasks, and align milestones with course deliverables and sponsor expectations.

Key artifacts:
Project Plan (PDF)
Gantt Chart / Schedule (Excel)

2. Requirements, Analysis & Design (RAD, SRS, SDD)

Next, we gathered and documented detailed functional and non-functional requirements, analyzed the client’s current processes, and designed a conceptual solution. The Software Requirements Specification (SRS) captures what the system must do, while the Software Design Document (SDD) describes how the system will be organized within the selected CRM platform, including key entities, relationships, and workflows.

Key artifacts:
Software Requirements Specification (SRS) (PDF)
Software Design Document (SDD) (PDF)

3. Development & Configuration

During the development phase, we implemented the proposed solution within HubSpot. This included configuring objects and properties, mapping spreadsheet fields into the CRM schema, setting up lists and pipelines, and preparing the environment for data import and future automation. The Development Document explains the configuration decisions and how they support the documented requirements.

Key artifact:
Development Document (PDF)

4. Prototype & In-Class Presentation

After initial configuration, we prepared a working prototype to demonstrate to the class and our sponsors. The prototype showed how data from Google Sheets could be translated and imported into HubSpot, how contacts and organizations are linked, and how the system supports the Georgia Laws of Life Essay Contest workflows.

Key artifact:
Prototype Presentation – Project Management Roadmap (PDF)

5. Testing (STP & STR)

We then developed a System Test Plan (STP) and executed test cases to verify that the configured solution met the documented requirements. The System Test Report (STR) summarizes test results, identifies any issues discovered during testing, and records evidence for how the system behaves under realistic usage scenarios.

Key artifact:
System Test Plan & System Test Report (STP & STR) (PDF)

6. Final Package & Client Handoff

Finally, we compiled the project results into a comprehensive final report and package for the client. This final package summarizes the problem context, requirements, design, development approach, testing activities, and our recommendations for the Georgia Laws of Life Essay Contest going forward. The project website you are viewing is included as part of that final deliverable.

Key artifact:
Final Report / Final Package (PDF)
